# WoOS Release Checklist (обязательно для каждого PR)

> В этом репозитории **каждый PR = новая версия WoOS**.

## Что обязан сделать автор PR
1. Поднять версию в файле `VERSION` (пример: `1.0.1 -> 1.0.2`).
2. Добавить запись в `CHANGELOG.md` с кратким списком изменений.
3. Обновить затронутую документацию (`DEVELOPMENT_PLAN.md`, технические заметки и т.д.).
4. Проверить, что версия отображается в UI/выводе системы (если применимо).
5. Убедиться, что сборка проходит (`make`).

## Что должен содержать PR
- Новый номер версии.
- Список изменений (что добавлено/изменено/исправлено).
- Обновления документации и пояснения для других разработчиков.
- Краткий план следующего шага (что делать в следующем PR).
