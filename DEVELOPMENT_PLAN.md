# WoOS Development Plan

> Цель: заложить оптимизированную и модульную базу ОС, чтобы постепенно прийти к
> расширяемой архитектуре, дружелюбному UI и фундаменту для будущей файловой системы.

## Принципы
- Модульность: изолировать подсистемы (boot, память, видео, UI, драйверы, FS).
- Минимальные зависимости: freestanding-код, простой ABI между модулями.
- Предсказуемый рендер: стабильный framebuffer pipeline без магии.
- Эволюционность: маленькие безопасные шаги с проверяемым результатом.

## Этап 0 — База запуска и видео (текущий)
- [x] Стабильный вход в long mode.
- [x] Передача параметров VBE framebuffer в ядро.
- [x] Базовый вывод графики в framebuffer.
- [x] Начало модульной структуры графического кода (`fb`, `ui`).

## Этап 1 — Базовый графический shell
- [x] Тема рабочего стола: фон + верхняя панель.
- [x] Примитивы рисования (заливка, прямоугольник, рамка).
- [ ] Bitmap-шрифт и текстовый рендер (статус/меню/часы).
- [ ] Буферизация/dirty-rect для оптимизации перерисовки.

## Этап 2 — Архитектурный фундамент ядра
- [ ] Простая модель модулей ядра (инициализация подсистем по этапам).
- [ ] Обработчики прерываний (IDT) и таймер (PIT/APIC).
- [ ] Менеджер физической памяти (PMM), затем виртуальной (VMM).

## Этап 3 — Подсистема хранения
- [ ] VFS-слой (единый интерфейс файловых операций).
- [ ] Базовая ФС (минимально: чтение каталогов/файлов).
- [ ] Дизайн собственной FS с журналированием и ACL-подобной моделью
      (в духе NTFS-подхода, но упрощённо).

## Этап 4 — UX и кастомизация
- [ ] Темы, конфиг интерфейса, layout-профили.
- [ ] Декоративные эффекты (тени/скругления/акценты) без тяжёлой композиции.
- [ ] Менеджер приложений/окон (долгосрочно).
